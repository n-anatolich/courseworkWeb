КТ №1: 14 апреля — Проектирование
Что должно быть готово
1. Выбор темы
* Утверждённая тема проекта из предложенных вариантов (или согласованная с руководителем)
* Краткое описание проекта (1-2 абзаца)
* Определение целевой аудитории
2. Анализ требований
* Список функциональных требований (что должен уметь делать сайт)
* Список не функциональных требований (скорость, безопасность, адаптивность)
* Описание ролей пользователей и их прав
3. Проектирование базы данных
* ER-диаграмма (можно нарисовать от руки и сфотографировать или в draw.io)
* Список таблиц с описанием полей:
   * Название таблицы
   * Названия полей, типы данных
   * Первичные и внешние ключи
   * Связи между таблицами
* Примеры данных для наполнения (минимум 5-10 записей)
4. Настройка окружения
* Установлен локальный сервер (OpenServer)
* Создана папка проекта
* Проверена работа PHP и MySQL
* Настроена база данных (создана, но пока пустая)
Что сдаётся преподавателю
1. Документ с описанием проекта (Word или Google Docs):
   * Название проекта
   * Цель и задачи
   * Описание функциональности
   * Список ролей
2. ER-диаграмма (скриншот или фото)
3. Структура БД (список таблиц и полей)
4. Подтверждение настройки окружения (скриншот phpMyAdmin c БД)
Советы
* ER-диаграмма поможет вам самим понять структуру, не пропускайте этот этап
* Если сомневаетесь в выборе темы, посоветуйтесь с преподавателем

КТ №2: 21 апреля — База данных
Что должно быть готово
1. SQL-скрипты
* Скрипт создания всех таблиц (CREATE TABLE ...)
* Скрипт вставки тестовых данных (INSERT INTO ...) — минимум:
   * 3-5 пользователей (включая администратора)
   * 10-20 записей в основной таблице (термины, материалы, карточки и т.д.)
   * 2-3 записи во вспомогательных таблицах (категории, единицы измерения)
2. Проверка БД
* Все таблицы созданы
* Внешние ключи работают (при попытке удалить связанную запись возникает ошибка или каскадное удаление)
* Тестовые данные загружены и отображаются
3. Файловая структура проекта
* Создана базовая структура папок:
* text
project/
├── index.php
├── admin/
├── includes/
│   ├── db.php
│   ├── header.php
│   └── footer.php
├── css/
├── js/
└── sql/
*     └── database.sql
4. Подключение к БД
* Написан файл db.php с подключением через PDO
* Проверено, что подключение работает (можно вывести сообщение об успехе)
Что сдаётся преподавателю
1. SQL-файл database.sql с полными скриптами создания и наполнения
2. Скриншот phpMyAdmin с отображением всех таблиц
3. Скриншот одной таблицы с данными (например, элементы или термины)
4. Демонстрация (устно) — что БД создана, данные загружены
Советы
* Сохраняйте SQL-скрипт в файл — пригодится для сдачи и для восстановления
* Используйте ON DELETE CASCADE для внешних ключей, чтобы упростить удаление
* Для паролей в тестовых данных используйте password_hash('123456', PASSWORD_DEFAULT) и вставьте хеш

КТ №3: 28 апреля — Базовая структура сайта и аутентификация
Что должно быть готово
1. Базовые страницы
* index.php — главная страница (заглушка с приветствием)
* header.php и footer.php — подключаются на всех страницах
* Простая навигация (ссылки на все разделы)
2. Аутентификация
* register.php — форма регистрации (username, email, password)
* login.php — форма входа
* logout.php — выход из системы
* Проверка: после регистрации пользователь появляется в БД
* Проверка: вход с правильным паролем работает
* Проверка: после входа в сессии появляются данные пользователя
3. Разграничение прав (базовое)
* На главной странице отображается разный контент для гостя и авторизованного
* Ссылка на админ-панель показывается только администратору
4. CSS
* Базовые стили (можно подключить Bootstrap)
* Формы выглядят аккуратно
Что сдаётся преподавателю
1. Демонстрация работы регистрации и входа
   * Показать форму регистрации
   * Показать, что после регистрации данные появились в БД
   * Показать вход и отображение имени пользователя
2. Код header.php и db.php (на экране или в репозитории)
3. Скриншот сессии (как вариант — показать в браузере)
Советы
* Не пытайтесь сделать всё сразу — сосредоточьтесь только на регистрации и входе
* Используйте password_hash() и password_verify()
* Проверьте, что сессии работают (session_start() в начале каждого файла)
* Для админ-панели пока хватит одной страницы-заглушки

КТ №4: 5 мая — CRUD для основной сущности
Что должно быть готово
1. Просмотр данных
* Страница со списком всех записей (терминов, материалов, карточек и т.д.)
* Пагинация (10-20 записей на страницу)
* Возможность просмотра детальной информации (отдельная страница)
2. Админ-панель (CRUD)
* Страница admin.php (доступна только администратору)
* Форма добавления новой записи
* Форма редактирования существующей записи
* Кнопка удаления (с подтверждением)
* После выполнения действий — перенаправление на список
3. Поиск и фильтрация (простая)
* Строка поиска по названию (через LIKE)
* Фильтр по категории (если есть)
4. Валидация
* Проверка, что обязательные поля заполнены
* Проверка, что значения корректны (например, атомный номер > 0)
Что сдаётся преподавателю
1. Демонстрация работы CRUD:
   * Добавление новой записи
   * Редактирование записи
   * Удаление записи
   * Просмотр списка
2. Демонстрация поиска — поиск по названию работает
3. Демонстрация админ-панели — только для администратора
Советы
* Сначала сделайте просмотр и добавление, потом редактирование и удаление
* Для редактирования используйте ту же форму, что и для добавления, с подставленными значениями
* Удаление делайте через GET или POST с подтверждением через JavaScript
* Проверяйте права на каждой странице админки

Что должно быть готово (в зависимости от варианта)

* Динамическая форма (поля меняются в зависимости от задачи)
* Расчёт по формулам
* Вывод пошагового решения

* Полнотекстовый поиск
* Фильтрация по категориям
* Алфавитный указатель
Для Варианта 5 (Конвертер величин)
* Реализовано 3-4 категории (длина, масса, время)
* Корректная конвертация через базовую единицу
* Конвертация температуры (отдельная формула)
Для Варианта 6 (Лабораторный журнал)
* Приход материалов (увеличение количества)
* Расход материалов (уменьшение количества)
* Проверка, что не списывается больше, чем есть

* Студент видит список своих работ
* Форма сдачи работы (текст + файл)
* Преподаватель видит работы своей группы
* Выставление оценки
Для Варианта 8 (Карточки)
* Показ карточки (формула → название)
* Проверка ответа (сравнение строк)
* Учёт правильных/неправильных ответов
* Простой алгоритм повторения
Что сдаётся преподавателю
1. Демонстрация основной функции (расчёт, конвертация, проверка ответа)
2. Примеры работы — показать 3-4 разных сценария
3. Обработка ошибок — показать, что происходит при неверном вводе
Советы
* Тестируйте на разных входных данных
* Добавьте понятные сообщения об ошибках
* Сохраняйте промежуточные результаты в БД (история)

Рекомендации по подготовке к каждой КТ
Общие рекомендации
1. Не откладывайте на последний день
2. Регулярно коммитьте в Git — если что-то сломается, всегда можно откатиться
3. Тестируйте сразу — после каждой новой функции проверяйте, что старое не сломалось
4. Ведите список задач — отмечайте, что сделано, что осталось
Что делать, если не успеваете
1. Сократите функционал — лучше сделать меньше, но качественно
2. Упростите задачу — например, вместо сложного парсера формул сделайте простой
3. Обратитесь к преподавателю — возможно, можно скорректировать требования
Что делать перед каждой КТ
1. Запустить проект — убедиться, что всё работает
2. Проверить базу данных — все ли таблицы на месте
3. Проверить права доступа — зайти под разными пользователями
4. Подготовить демо — знать, что показывать
