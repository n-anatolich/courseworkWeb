Вариант 3: «Физический калькулятор: законы Ньютона и механика»
1. Общая постановка задачи
Разработать веб-приложение для решения задач по классической механике. Пользователь выбирает тип задачи (кинематика, динамика, работа и энергия), вводит известные величины, система вычисляет неизвестные параметры, отображает формулу и пошаговое решение. Для зарегистрированных пользователей сохраняется история расчётов. Администратор управляет справочником физических констант и может добавлять новые типы задач.

	1.5

	2.3. Модули системы
Модуль 1: Справочник физических констант
            * Таблица констант с возможностью поиска
            * Для каждой константы: название, символ, значение, единица измерения
            * Администратор может добавлять, редактировать, удалять константы
            * Константы по умолчанию:
            * g = 9.8 м/с² (ускорение свободного падения)
            * G = 6.67430×10⁻¹¹ Н·м²/кг² (гравитационная постоянная)
            * π = 3.14159
Модуль 2: Калькулятор задач
            * Выбор категории (кинематика, динамика, работа и энергия и т.д.)
            * Выбор конкретной задачи из выбранной категории
            * Динамическая форма ввода (поля появляются в зависимости от задачи)
            * Кнопка «Рассчитать»
            * Вывод:
            * Результат с единицами измерения
            * Формула (в читаемом виде, с использованием MathJax или текстом)
            * Пошаговое решение
            * Схематическое изображение (опционально, текстовое описание)
Модуль 3: Конструктор задач (для авторизованных)
            * Пользователь может создавать свои задачи с произвольными параметрами
            * Сохранение в личном кабинете
            * Возможность сделать задачу публичной (доступной другим пользователям)
Модуль 4: История расчётов
            * Автоматическое сохранение каждого успешного расчёта:
            * Дата и время
            * Тип задачи
            * Входные параметры (в JSON)
            * Результат
            * Просмотр истории с пагинацией
            * Возможность удалить расчёт
            * Возможность повторить расчёт (кнопка «Использовать снова»)
Модуль 5: Административная панель
            * Управление константами (CRUD)
            * Управление типами задач (CRUD) — можно добавлять новые формулы
            * Управление пользователями (блокировка/разблокировка)
            * Статистика использования:
            * Количество расчётов по типам задач
            * Самые популярные задачи
            * Активность пользователей
Модуль 6: Аутентификация
            * Регистрация: username, email, пароль
            * Вход: username или email + пароль
            * Выход

3. Технические требования
3.1. Клиентская часть
            * HTML5 — семантическая вёрстка
            * CSS3 — адаптивный дизайн, поддержка мобильных устройств
            * JavaScript (ES6+):
            * Динамическое изменение формы ввода при выборе типа задачи
            * Валидация ввода (числа, положительные значения, допустимые диапазоны)
            * Асинхронные запросы к серверу
            * Отображение формул (можно использовать MathJax или простой текст)
            * Сохранение состояния формы в LocalStorage
3.2. Серверная часть
            * PHP 8.x — основная логика
            * PDO — работа с базой данных
            * Модуль вычислений — отдельный класс для каждого типа задачи
3.3. База данных (MySQL)
sql
-- Таблица пользователей
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    role ENUM('user', 'admin') DEFAULT 'user',
    is_blocked BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);


-- Таблица физических констант
CREATE TABLE constants (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    symbol VARCHAR(20) NOT NULL,
    value DECIMAL(20,10) NOT NULL,
    unit VARCHAR(50) NOT NULL,
    description TEXT
);


-- Таблица категорий задач
CREATE TABLE problem_categories (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    sort_order INT DEFAULT 0
);


-- Таблица типов задач
CREATE TABLE problem_types (
    id INT AUTO_INCREMENT PRIMARY KEY,
    category_id INT NOT NULL,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    formula_text TEXT,  -- текст формулы для отображения
    formula_expression TEXT,  -- выражение для вычисления
    input_fields JSON,  -- описание полей ввода
    output_fields JSON,  -- описание полей вывода
    sort_order INT DEFAULT 0,
    FOREIGN KEY (category_id) REFERENCES problem_categories(id) ON DELETE CASCADE
);


-- Таблица истории расчётов
CREATE TABLE calculations (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    problem_type_id INT NOT NULL,
    input_data JSON NOT NULL,
    result_data JSON NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (problem_type_id) REFERENCES problem_types(id)
);


-- Таблица пользовательских задач
CREATE TABLE user_problems (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    name VARCHAR(200) NOT NULL,
    description TEXT,
    problem_type_id INT NOT NULL,
    input_data JSON NOT NULL,
    result_data JSON NOT NULL,
    is_public BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (problem_type_id) REFERENCES problem_types(id)
);


-- Индексы для быстрого поиска
CREATE INDEX idx_calculations_user ON calculations(user_id);
CREATE INDEX idx_calculations_date ON calculations(created_at);
CREATE INDEX idx_user_problems_user ON user_problems(user_id);

4. Примеры расчётов
Пример 1: Равномерное движение
Вход:
            * Тип задачи: Равномерное движение
            * Скорость (v) = 10 м/с
            * Время (t) = 5 с
Выход:
text
Расстояние (s) = 50 м


Решение:
1. Формула: s = v × t
2. Подставляем значения: s = 10 × 5
3. Получаем: s = 50 м
Пример 2: Второй закон Ньютона
Вход:
            * Тип задачи: Второй закон Ньютона
            * Масса (m) = 10 кг
            * Ускорение (a) = 2 м/с²
Выход:
text
Сила (F) = 20 Н


Решение:
1. Формула: F = m × a
2. Подставляем значения: F = 10 × 2
3. Получаем: F = 20 Н
Пример 3: Свободное падение
Вход:
            * Тип задачи: Свободное падение
            * Время падения (t) = 3 с
            * g = 9.8 м/с² (из справочника)
Выход:
text
Высота (h) = 44.1 м
Скорость в момент падения (v) = 29.4 м/с


Решение:
1. Формула: h = g × t² / 2
2. Подставляем: h = 9.8 × 9 / 2 = 44.1 м
3. Формула: v = g × t
4. Подставляем: v = 9.8 × 3 = 29.4 м/с
Пример 4: Абсолютно неупругий удар
Вход:
            * Тип задачи: Абсолютно неупругий удар
            * Масса первого тела (m₁) = 2 кг
            * Скорость первого тела (v₁) = 3 м/с
            * Масса второго тела (m₂) = 3 кг
            * Скорость второго тела (v₂) = 1 м/с (навстречу, знак учтён)
Выход:
text
Общая скорость после удара (u) = 1.8 м/с


Решение:
1. Формула: u = (m₁·v₁ + m₂·v₂) / (m₁ + m₂)
2. Подставляем: u = (2 × 3 + 3 × 1) / (2 + 3)
3. u = (6 + 3) / 5 = 9 / 5 = 1.8 м/с

5. Возможные расширения (для повышенной оценки)
            1. Визуализация — простые схемы движения (Canvas)
            2. Векторный ввод — возможность указывать направление сил и скоростей
            3. Графики зависимости — построение графиков v(t), s(t), a(t)
            4. Экспорт — сохранение расчётов в PDF
            5. Интерактивные тесты — проверка знаний формул
            6. Решение систем уравнений — для задач с несколькими неизвестными
            7. Поддержка единиц измерения — автоматический перевод (км/ч → м/с)
________________


6. Рекомендации по реализации
            1. Начните с простого: реализуйте 3-4 базовые задачи (равномерное движение, второй закон Ньютона, работа)
            2. Используйте единый класс-калькулятор с методами для каждого типа задач
            3. Храните формулы в БД как JSON, но для простоты можно захардкодить первые 5-10 задач
            4. Сделайте удобный выбор задачи — выпадающий список с категориями
            5. Добавьте подсказки — какие поля нужно заполнить для расчёта
            6. Проверяйте деление на ноль и отрицательные значения
